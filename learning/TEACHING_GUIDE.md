# Agents from Scratch 教学指南

## 1. 教学目标

帮助学习者理解 Agent 是如何围绕 LLM 被工程化出来的，而不是让学习者只会复制框架 API。课程结束时，学习者应能追踪数据、识别信任边界，并搭建一个小型可靠 Agent。

## 2. 适用学习者

- 能使用终端和编辑 Python 文件；
- Python 基础可以不完整；
- 不要求具备机器学习模型实现经验；
- 适合通过手敲、预测、运行和观察具体值学习。

Python 语法随 Agent 代码按需引入：函数、返回值、列表、字典、类、对象、import、异常、循环、集合、回调和类型提示。

## 3. 固定教学流程

每一课遵循以下步骤：

1. **提出问题**：只说明本课解决的一个工程问题；
2. **先做预测**：预测输出、状态变化和失败位置；
3. **阅读文档**：翻译并解释 `lessons/XX_*.md`；
4. **寻找入口**：定位 `complete_example.py` 对应函数；
5. **追踪公开方法**：进入 `agent/agent.py`；
6. **按需读 helper**：只继续阅读当前数据流需要的模块；
7. **从零手敲**：完成一个独立的最小练习；
8. **运行观察**：打印 raw output 和 parsed data；
9. **划分职责**：说明模型与 Python 各自做了什么；
10. **观察失败**：主动触发或分析至少一种失败；
11. **脱稿复述**：不看代码说明完整调用链；
12. **记录结果**：更新进度，完成后单独 Git 提交。

不要一开始解释整个仓库。每次交流只聚焦一个新能力和一个必要的 Python 概念。

## 4. 教学模块

### Module A：模型接口，Lesson 01-03

依次教授文本生成、行为提示词和结构化契约。必须让学习者理解：JSON 起初是模型生成的字符串，只有解析成功后才成为 Python 字典。

### Module B：最小 Agent，Lesson 04-06

教授有限路由、工具请求和 observe-decide-act 循环。模型只请求动作，Python 负责授权和执行；循环只有获得新 observation 才会真正推进。

### Module C：任务系统，Lesson 07-10

教授模型外 memory、可检查的 plan、带参数的 atomic action 和依赖图。持续区分模拟执行与真实副作用。

### Module D：可靠性，Lesson 11-12

教授 Evals 和 Telemetry。Evals 回答“行为是否仍满足契约”，Telemetry 回答“某一次运行到底发生了什么”。

### Module E：综合项目

连接工具、observation、state、memory、planning、evals 和 telemetry。只有学习者理解了这些部件后，再介绍框架提供了哪些抽象。

## 5. 每课必须回答的四个问题

1. 本课新增了什么能力？
2. 公开代码入口在哪里？
3. 哪些工作属于模型，哪些属于 Python？
4. 会在哪里失败，如何检测？

## 6. 用具体值教授数据流

不要只讲抽象变量，始终代入具体值：

```text
input = "Translate hello"
expected = "translate"
model text = {"decision": "translate"}
actual = "translate"
passed = actual == expected
```

然后追踪这些值如何进入函数参数、局部变量、返回值、列表、字典和最终输出。

## 7. 常见误区

| 误区 | 正确认知 |
| --- | --- |
| 模型文件会自己运行 | GGUF 保存模型数据，llama.cpp 执行推理 |
| `Llama` 就是模型 | 它是加载模型文件的 Python 接口对象 |
| LLM 会记住上次运行 | 必须在外部保存并重新提供 memory |
| JSON 输出已经是 Python 字典 | 解析前仍然只是字符串 |
| 选择工具就等于执行工具 | 模型提出请求，Python 校验并执行 |
| 生成计划就完成了任务 | plan 只是数据，executor 才产生真实效果 |
| 循环次数增加就是有进展 | 必须改变状态或获得新 observation |
| Eval 失败说明 Eval 坏了 | 也可能正确发现 Agent 回归，应查看 `error` 和 `actual` |

## 8. 学习验收

- **识别**：能指出某项能力属于哪个函数；
- **追踪**：能预测关键代码行执行后的具体值；
- **迁移**：改变输入或契约后，能解释结果为什么变化。

一课达到可教授程度的标准是：能画调用链、能解释一个失败，并能设计一个最小 Eval。

## 9. 教学边界

- 不替学习者输入所有命令或完成所有编辑；
- 在展示 raw output 前，不用抽象隐藏模型输出；
- 不把模拟返回值称为真实执行；
- 不把一次成功输出当作可靠性证据；
- 不提前用框架掩盖尚未理解的概念；
- 模型可请求的权限必须小于 Python 实际可用能力。

## 10. 推荐单次课程节奏

```text
10 分钟  复习与预测
15 分钟  文档阅读与源码追踪
30 分钟  从零手敲练习
15 分钟  运行、调试与观察数据
10 分钟  脱稿复述、Eval 和学习记录
```
